| 110 | 3 | **[VA-07] `RecipeAnimator` MonoBehaviour — main channel execution** | Runs a `Recipe` sequentially: for each `Phase`, execute all steps in parallel via coroutines, then wait for all to finish before advancing. Exposes `Play(Recipe)` and `Stop()`. |
| 110 | 2 | **[VA-08] Additive overlay channel in `RecipeAnimator`** | Second, independent coroutine slot running alongside the main channel. Used for hit-react and charge-up overlays. `PlayOverlay(Recipe)` interrupts any running overlay and starts the new one. |

Ensure resolution is not skipped if the animator is missing. |

| 95 | 2 | **[VA-13] `MoveLocalStep`, `RotateLocalStep`, `ScaleStep` concrete steps** | Each: `[SerializeReference]` concrete `AnimationStep` subclass. Fields: target `LogicalPart`, target value (Vector3), duration, easing curve. Executed by `RecipeAnimator` as a timed lerp coroutine. |
| 95 | 2 | **[VA-14] `ShakeStep`, `FlashStep`, `WaitStep` concrete steps** | `ShakeStep`: random offset applied to a `LogicalPart` for a duration. `FlashStep`: toggles renderer visibility at a set interval. `WaitStep`: no-op that simply waits for a duration — useful for spacing phases. |
| 95 | 2 | **[VA-15] `SpawnVfxStep`, `PlaySoundStep` concrete steps — stub** | Stub implementations: `SpawnVfxStep` holds a prefab reference and a target `LogicalPart`; `PlaySoundStep` holds an `AudioClip`. Execution is empty until VA-18 and VA-19 implement the actual behaviour. |

| 90 | 1 | **[VA-17] Map `LogicalPart` to concrete `Transform` in `AnimationRig`** | `AnimationRig.GetTransform(LogicalPart) → Transform`. Returns the mapped transform, or the root transform as fallback if the part is not mapped. Logs a warning on missing mappings. |
